准备工作

搭建前需要注册账号、安装 CLI 工具和配置 AI 辅助 Skills，大约 10 分钟。

需要注册的账号

1
GitHub 账号 代码托管平台，存放博客源码、触发自动部署 → github.com
2
Vercel 账号 建议直接用 GitHub 登录，免去绑定步骤 → vercel.com
3
域名（可选） 自定义博客地址，推荐 Cloudflare / Namesilo / 阿里云

本地工具安装

  • 安装 Git — 版本控制核心工具 — git-scm.com
  • 安装 Node.js 18+ — 运行 Next.js 和包管理 — nodejs.org
  • 安装编辑器 VS Code（推荐）
  • 配置 Git 用户信息
Terminal — 配置 Git 
# 配置 Git 用户名和邮箱
git config --global user.name "你的名字"
git config --global user.email "your@email.com"

# 验证安装
git --version
node --version

CLI 工具安装与用途

以下命令行工具是整个工作流的核心，每个都有明确用途：

Git CLI — 版本控制核心

用途：管理代码版本、提交文章、推送到 GitHub 触发自动部署。

安装：下载 git-scm.com 或 winget install Git.Git

# 核心命令
git init                    # 初始化仓库
git add .                    # 暂存所有更改
git commit -m "msg"          # 提交到本地
git push origin main         # 推送 → 触发 Vercel 自动部署
git pull                     # 拉取远程更新
Vercel CLI — 部署与本地预览

用途：本地预览生产构建、手动部署、管理环境变量、查看部署日志。

# 安装
npm i -g vercel

# 核心命令
vercel                     # 部署到 Vercel（交互式）
vercel dev                  # 本地模拟 Vercel 环境
vercel env pull             # 拉取远程环境变量到本地
vercel ls                   # 查看所有部署
vercel inspect <url>        # 查看部署详情与性能
vercel domains add xxx.com  # 添加自定义域名
npm / npx — 包管理

用途：安装项目依赖、运行开发服务器、执行脚手架命令。随 Node.js 一起安装。

npm install                  # 安装项目依赖
npm run dev                  # 启动本地开发服务器 (localhost:3000)
npm run build                # 生产构建（检查能否通过编译）
npx create-next-app@latest   # 用最新版创建 Next.js 项目
GitHub CLI (gh) — GitHub 操作（可选）

用途：命令行直接操作 GitHub：创建仓库、PR、Issue，不用打开浏览器。

# 安装
winget install GitHub.cli

# 核心命令
gh auth login                # 登录 GitHub
gh repo create my-blog --public  # 命令行创建仓库
gh pr create                 # 创建 Pull Request
gh pr list                   # 查看 PR 列表

部署上线

两种部署方式：Vercel 自动部署（推荐）或 GitHub Pages 静态托管。

Step 1: 创建仓库并推送

用 gh CLI 一条命令创建远程仓库，免去手动去网页操作：

Terminal — gh CLI 全流程 
# 1. 初始化本地仓库
git init
git add .
git commit -m "init: first commit"

# 2. 用 gh CLI 直接创建 GitHub 公开仓库（不用开浏览器）
gh repo create my-blog --public --description "我的个人博客"

# 3. 推送到 GitHub
git branch -M main
git remote add origin https://github.com/你的用户名/my-blog.git
git push -u origin main
gh 的优势：一条命令完成创建仓库 + 设置 remote + 推送。也可以用 gh repo create --source=. --push 一步到位，自动关联当前目录并推送。

Step 2A: Vercel 部署（推荐）

1
登录 Vercel Dashboard 打开 vercel.com 并用 GitHub 登录
2
点击 "Add New" → "Project" 选择 "Import Git Repository"
3
选择你的 my-blog 仓库 Vercel 会自动检测 Next.js 框架，直接点 Deploy
4
等待部署完成 约 1~2 分钟，你会获得一个 xxx.vercel.app 地址
自动部署：之后每次 git push，Vercel 会自动检测并重新部署，无需手动操作！这就是 CI/CD 的魅力。

Step 2B: GitHub Pages 部署（纯静态）

如果博客是纯 HTML/CSS 或静态导出文件，可以直接用 GitHub Pages 免费托管：

Terminal — GitHub Pages 开启 
# 方式 1: 用 gh CLI 一键开启 GitHub Pages
gh api repos/{owner}/{repo}/pages \
  -X POST -f build_type=workflow

# 方式 2: 在仓库 Settings → Pages → Source 选 main 分支
# 访问地址: https://你的用户名.github.io/my-blog/
Next.js 导出静态文件用于 GitHub Pages

Next.js 可以导出纯静态文件，然后托管到 GitHub Pages：

# next.config.js 添加
output: 'export'

# 构建静态文件
npm run build
# 生成 out/ 目录，直接部署到 GitHub Pages

# 或用 gh-pages 分支自动部署
npx gh-pages -d out

写文章 — Git 工作流

用 Markdown 写作，Git 管理版本，推送即发布。

创建一篇文章

在 content/ 目录下创建 Markdown 文件：

content/my-first-post.md 
---
title: "我的第一篇博客"
date: "2025-01-15"
excerpt: "这是摘要内容"
tags: ["随笔", "技术"]
---

## Hello World

这是我的第一篇文章，使用 Markdown 编写。

### 支持的语法

- **粗体** 和 *斜体*
- 代码块: `console.log('hello')`
- 图片: ![描述](/images/photo.jpg)
- 链接: [文字](https://example.com)

Git 发布流程

Terminal — 每次发布文章 
# 1. 写完文章后，添加到暂存区
git add content/my-first-post.md

# 2. 提交
git commit -m "post: 添加第一篇博客文章"

# 3. 推送 → Vercel 自动部署！
git push origin main
提交规范：建议使用约定式提交：post: 新文章、fix: 修复、feat: 新功能。方便日后回顾。
高级工作流：草稿分支 + PR 预览

对于重要的文章，可以先创建草稿分支，写完再通过 Pull Request 合并。Vercel 会为每个 PR 自动生成预览链接，方便你检查效果。

# 创建草稿分支
git checkout -b draft/my-article

# 写完后推送并创建 PR
git push origin draft/my-article
# 在 GitHub 上创建 Pull Request
# Vercel 会自动生成预览 URL

绑定自定义域名

让你的博客拥有专属域名，如 blog.yourname.com。

1
购买域名 推荐 Namesilo / Cloudflare / 阿里云，一般 .com 域名年费 ≈ ¥50-70
2
Vercel 添加域名 Settings → Domains → 输入你的域名
3
配置 DNS 解析 在域名商后台添加 CNAME 记录指向 cname.vercel-dns.com
4
等待生效 DNS 传播需要几分钟到几小时，Vercel 自动配置 HTTPS 证书
HTTPS 免费自动配置：Vercel 会自动为你的域名签发 SSL 证书，无需额外操作。

    常见问题 FAQ

    新手最常遇到的问题，点击展开查看解答。

    Vercel 免费版有什么限制？

    免费额度：每月 100GB 带宽、无限次部署、Serverless 函数 10秒超时。个人博客完全够用，日均几千访问量没有问题。

    推送后多久能看到更新？

    通常 30 秒 ~ 2 分钟。Vercel 检测到 push 后会自动触发构建和部署。如果文章较多，构建时间会长一些。

    不想用 Next.js，有其他选择吗？

    可以！Vercel 支持多种框架：Astro（推荐静态博客）、Hugo、Hexo、Gatsby、甚至纯 HTML。Astro 特别适合博客，默认零 JS 输出。

    如何在手机上写文章？

    可以用 GitHub 网页端直接编辑 Markdown 文件并 commit。或者使用 StackEdit、Obsidian 等工具写好后粘贴到 GitHub。

    部署失败了怎么办？

    在 Vercel Dashboard → Deployments 中查看构建日志，常见原因：依赖安装失败（检查 package.json）、语法错误、环境变量缺失。
